# Git 応用演習

## 目的

この演習では、Git の応用的な機能を実践的に学びます。ブランチ、マージ、リモートリポジトリの操作など、チーム開発で必要となる高度な Git コマンドを体験します。

## 前提条件

- Git 基礎演習（`1_basic/1_Exercise`）を完了していること
- 基本的な Git コマンド（`git add`, `git commit`, `git status`など）を理解していること

## 演習の流れ

### 準備

1. このディレクトリで新しい Git リポジトリを初期化します
2. `Challenge`フォルダ内の HTML と CSS ファイルを使って演習を進めます
3. 各ステップで Git コマンドを実行し、ブランチやマージなどの操作を体験します

### 学習内容

- `git branch`: ブランチの作成と管理
- `git checkout` / `git switch`: ブランチの切り替え
- `git merge`: ブランチのマージ
- `git remote`: リモートリポジトリの管理
- `git push` / `git pull`: リモートとの同期
- `git rebase`: コミット履歴の整理
- `git stash`: 一時的な変更の保存
- コンフリクトの解決方法

## 完了後

すべてのチェックリストを完了したら、`Answer`フォルダの解説を確認して理解を深めましょう。

## ヒント

- ブランチは機能開発の独立した作業ラインです
- マージ前に必ず`git status`で状態を確認しましょう
- コンフリクトが発生した場合は落ち着いて解決しましょう
- リモートリポジトリは GitHub などのサービスを利用できます
