## チェックリスト

> **注意**: このリポジトリはすでに初期化されています。`Challenge`フォルダに移動して、すぐに演習を始められます。

### ステップ 1: 最初のコミット

- [ ] `git add index.html`で`index.html`をステージングする
- [ ] `git add style.css`で`style.css`をステージングする
- [ ] `git status`でステージング状態を確認する
- [ ] `git commit -m "初回コミット: HTMLとCSSファイルを追加"`でコミットする
- [ ] `git log`でコミット履歴を確認する

### ステップ 2: HTML の修正

- [ ] `index.html`を開き、`<h1>`タグの内容を「Git ワークショップへようこそ！」に変更する
- [ ] `git status`で変更を確認する
- [ ] `git diff`で具体的な変更内容を確認する
- [ ] `git add index.html`で変更をステージングする
- [ ] `git commit -m "タイトルを更新"`でコミットする

### ステップ 3: CSS の修正

- [ ] `style.css`を開き、背景色を`#e3f2fd`に変更する
- [ ] `git status`で変更を確認する
- [ ] `git diff style.css`で変更内容を確認する
- [ ] `git add style.css`でステージングする
- [ ] `git commit -m "背景色を青系に変更"`でコミットする

### ステップ 4: 複数ファイルの同時変更

- [ ] `index.html`に新しい段落（`<p>`タグ）を追加する
- [ ] `style.css`でフォントサイズを調整する
- [ ] `git status`で複数ファイルの変更を確認する
- [ ] `git add .`ですべての変更をステージングする
- [ ] `git commit -m "コンテンツとスタイルを追加"`でコミットする

### ステップ 5: コミット履歴の確認

- [ ] `git log`でこれまでのコミット履歴を確認する
- [ ] `git log --oneline`で簡潔な履歴を確認する
- [ ] `git log --graph --oneline`でグラフ表示を確認する

### ステップ 6: 変更の取り消し（オプション）

- [ ] `index.html`に適当な変更を加える
- [ ] `git status`で変更を確認する
- [ ] `git restore index.html`で変更を取り消す
- [ ] `git status`で元に戻ったことを確認する

## 完了後

すべてのチェックリストを完了したら、`Answer`フォルダの解説を確認して理解を深めましょう。

## ヒント

- コマンドがわからなくなったら`git --help`や`git <コマンド> --help`で確認できます
- `git status`は何度実行しても問題ありません。迷ったら実行しましょう
- コミットメッセージは変更内容がわかるように具体的に書きましょう
